Command Reference

Type any command in the Chat tab or use the menu bar above. Commands are case-insensitive. Natural language also works — just describe what you need.

🖥 System Checks
CommandDescriptionExample
check diskDisk usage — total, used, free GB and % used.check disk on prod
check memoryRAM usage in MB with total, used, and % used.check memory on all
check cpuCurrent CPU usage % and logical core count.
check uptimeHow long the server has been running.
check portsAll open listening TCP/UDP ports.
top processesTop 5 processes by CPU usage.
system healthFull health summary — disk, memory, CPU, uptime all in one.system health on all
check failed servicesLists systemd services currently in a failed state.
🌐 Network
CommandDescriptionExample
check ipNetwork interfaces with assigned IPs (IPv4 and IPv6).
check routesRouting table: destination, gateway, interface, metric.
check networkPer-interface stats: bytes sent/received, packets.
check dnsTests DNS resolution for google.com, cloudflare.com, github.com.
check connectionsCount of currently established TCP connections.
⚙️ Services
CommandDescriptionExample
service status <name>Checks if a systemd service is active, inactive, or failed.service status nginx
restart <name>Initiates a service restart — always requires confirmation.restart nginx
confirm restart <name>Confirms and executes the pending service restart.confirm restart nginx
cancelCancels any pending confirmation (restart, runbook, SSH key).
🖥 Multi-Node
CommandDescriptionExample
add node <name> <user>@<host>Registers a new remote node with a short alias.add node prod ubuntu@10.0.0.5
list nodesShows all registered nodes — name, host, SSH user, key.
remove node <name>Removes a node from the registry.
copy ssh key <user>@<host>Copies the ChatOps SSH public key to a remote host.
<cmd> on <node>Runs any command on a specific registered node via SSH.check disk on prod
<cmd> on allRuns across all registered nodes simultaneously.system health on all
📋 Logs, Alerts & Runbooks
CommandDescriptionExample
analyze logs: <text>AI analysis of pasted log content — or use Upload Log button.Or click Upload Log
show alerts5 most recent alerts.show alerts 20
show critical alertsCritical alerts only.
show unacked alertsUnacknowledged alerts only.
list runbooksLists all available runbooks.
run <runbook>Initiates a runbook — requires confirmation before running.run clear_tmp
show test logsLists all saved test log files with AI analysis option.
run testsRuns pytest in background and saves log. Returns immediately.
🎫 ITSM Tickets
CommandDescriptionExample
show ticketsLists open tickets.
show all ticketsLists all tickets regardless of status.
show ticket <id>Full details for a specific ticket.show ticket 3
create ticketInteractively create a new ticket.
close ticket <id>Closes a ticket by ID.close ticket 3
link ticket <id> <alert_id>Links a ticket to an alert.
rca <alert_id>AI-generated Root Cause Analysis draft for an alert.rca 42
📈 Analytics
CommandDescriptionExample
show analyticsAlert stats, MTTR trend chart, and team leaderboard.show analytics 14
show prometheus metricsPrometheus-format metrics output with AI analysis option.
analyze prometheusAI interpretation of Prometheus metrics with suggested actions.
configure prometheusShows which metrics are enabled / disabled in the export.
download pdf reportGenerates and downloads a PDF analytics report.download pdf report 14
show report24h health report for all nodes in chat.
💬 Slack & Reports
CommandDescriptionExample
config set slack_webhook <url>Sets the Slack incoming webhook URL.
test slackSends a test alert to the configured Slack webhook.
config set alert suppress <min>Sets the alert suppression window (default 10 min).config set alert suppress 30
config set report on/offEnables / disables the daily Slack report.
config set report hour <0-23>Sets the hour for the daily Slack report.
👥 User Management
CommandDescriptionExample
list usersLists all users, roles, and creation date.
add user <name> <pass> <role>Creates a new user. Roles: viewer, operator, developer, admin.add user alice pw123 operator
remove user <name>Deletes a user account.
change password <pass>Changes your own password.
show audit logShows last 20 audit log entries.show audit log 50
⚡ General
CommandDescriptionNotes

💡 Natural Language

You can also describe what you want in plain English. For example: "how much disk space is left", "is nginx running", or "run rca on alert 5" — the AI will map it to the right command automatically.
