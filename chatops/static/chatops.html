📚 Command Reference

Type any command in the Chat tab. Commands are case-insensitive. Use help in chat for a quick summary.

System Checks
CommandWhat it doesExample / Notes
check diskShows disk usage — total, used, free GB and % used. Status badge: OK / WARNING / CRITICAL based on configured thresholds.check disk
Also: check disk on newnode
check memoryShows RAM usage in MB with total, used, and % used. Status badge included.check memory
Also: check memory on all
check cpuShows current CPU usage % and logical core count. Status badge included.check cpu
check uptimeShows how long the server has been running in days, hours, minutes.check uptime
Also: check uptime on newnode
check portsLists all open listening TCP/UDP ports on the server.check ports
Also: check ports on newnode
top processesLists top 5 processes sorted by CPU usage, showing PID, name, CPU%, and memory%.top processes
Also: top processes on newnode
system healthFull health summary — disk, memory, CPU, and uptime all in one response. Each metric shows OK / WARNING / CRITICAL. Use to get a quick overall picture of any node.system health
system health on newnode
system health on all
check failed servicesLists systemd services currently in a failed state. Useful for spotting crashed daemons.check failed services
Also: check failed services on newnode

Services
CommandWhat it doesExample / Notes
service status <name>Checks if a systemd service is active, inactive, or failed. Replace <name> with the service name.service status nginx
service status ssh
restart <name>Initiates a service restart — always requires confirmation first. Shows a safety warning if restarting SSH/sshd (will drop your session).restart nginx
Then: confirm restart nginx
confirm restart <name>Confirms and executes the pending service restart. Must match the service name used in the restart command.confirm restart nginx
Use cancel to abort.
check failed servicesLists all systemd services currently in a failed state using systemctl --failed.check failed services
Multi-Node
CommandWhat it doesExample / Notes
add node <name> <user>@<host>Registers a new remote node. The name is a short alias you'll use in commands. Requires SSH key access already set up on the remote host.add node prod ubuntu@10.0.0.5
add node newnode user@1.2.3.4
list nodesShows all registered nodes — name, host, SSH user, and key path. The local ChatOps server is always listed first.list nodes
remove node <name>Removes a node from the registry. Existing metric history for that node is kept in the database.remove node prod
copy ssh key <user>@<host>Copies the ChatOps server's public SSH key to the remote host so passwordless SSH works. Auto-generates an SSH key pair if one doesn't exist yet. Requires confirmation.copy ssh key ubuntu@10.0.0.5
Then: confirm copy ssh key
confirm copy ssh keyConfirms and executes the pending SSH key copy. The remote host must allow password authentication for the initial key copy.confirm copy ssh key
Use cancel to abort.
<command> on <node>Runs any supported command on a specific registered node via SSH. The node name must match one registered with add node.check disk on newnode
system health on prod
check ports on newnode
<command> on allRuns the command across all registered nodes simultaneously and returns results for each. The local server is included automatically.system health on all
check disk on all
Logs, Alerts & Runbooks
CommandWhat it doesExample / Notes
analyze logs: <content>Paste log content after the colon to get an instant analysis: severity level, root cause, impact, and suggested actions. Alternatively, use the Upload Log button to analyze a file.analyze logs: ERROR db connection refused...
Or click Upload Log button.
show alertsShows the 5 most recent alerts from all nodes. Filter by node to see only that node's alerts. Acknowledged alerts can be managed in the Alerts tab.show alerts
show alerts on newnode
list runbooksLists all available runbooks — pre-defined automated remediation scripts stored on the server.list runbooks
run <runbook>Initiates a runbook execution — always asks for confirmation before running. Use the exact name shown in list runbooks.run disk-cleanup
Then: confirm disk-cleanup
confirm <runbook>Confirms and executes the pending runbook. Must be run within a short window after the run command.confirm disk-cleanup
cancelCancels any pending confirmation — works for service restarts, runbooks, and SSH key copies.cancel
Slack & Reports
CommandWhat it doesExample / Notes
config set slack_webhook <url>Sets the Slack incoming webhook URL. One webhook applies to all nodes — CRITICAL alerts from any node are sent to this URL. The URL is case-sensitive.config set slack_webhook https://hooks.slack.com/services/...
test slackSends a test alert to the configured Slack webhook to verify the integration is working. Reports the actual HTTP error if it fails.test slack
config set alert suppress <minutes>Sets the alert suppression window. If a metric stays CRITICAL, Slack is only notified once per window (per node per metric). Default is 10 minutes.config set alert suppress 30
Prevents Slack spam during ongoing incidents.
config set report onEnables the daily automated Slack report. The report covers all nodes and posts at the configured hour.config set report on
config set report offDisables the daily automated Slack report.config set report off
config set report hour <0–23>Sets the hour (24h, server time) at which the daily report posts to Slack. Also enables the report automatically.config set report hour 9 → 9:00 AM
config set report hour 18 → 6:00 PM
show reportGenerates and displays the last 24h health report for all nodes immediately in the chat — avg, min, max per metric and alert counts per node.show report
General
CommandWhat it doesExample / Notes
show system configShows the full system configuration — all thresholds (disk, memory, CPU), health check interval, Slack webhook, alert suppression window, and daily report settings.show system config
Also: config

ⓘ UI-only settings

Alert thresholds (disk / memory / cpu — warning & critical levels) and per-node thresholds are configured in the Config tab. They cannot be changed via chat commands.
